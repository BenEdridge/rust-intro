Getting started with rustup

The toolchain installer and go to for cross compilation, version changes, profile management, plugins etc.

The nvm or n version manager of Rust but more powerful

brew install rustup-init

For Other Methods and GPG verification

The package manager cargo and package registry crates.io

cargo is the package manager for Rust that pulls in packages from crates.io and provides many similar functions to npm.

A simple registry can be implemented in Git.

cargo provides a number of commands for building, cleaning, installing and publishing packages.

Some of the commands we will be using are:

#Create a new cargo package
cargo init

#Run a binary or example of the local package
cargo run

#Build this package's and its dependencies' documentation
cargo doc

#Run the tests
cargo test

The linter, rust-clippy and code formatter rustfmt

rustup gives the possibility to install a linter and code formatter providing functions similar to eslint and prettier in JavaScript

Linting by default with rustc https://doc.rust-lang.org/rustc/lints/groups.html

rustdoc and mdbook

Rust source code supports /// comment annotations to generate full blown documentation. mdbook creates a full book from markdown and code annotations.

/// You can have rust code between fences inside the comments
/// If you pass --test to `rustdoc`, it will even test it for you!
/// ```
/// use doc::Person;
/// let person = Person::new("name");
/// ```

Debugging with gbd and lldb

Rust provides debugging symbols when compiling so any native debugger will work.

Both VSCode and IntelliJ have plugin support.

rustc (:P), println!, dbg!, gdb, rr - in that order.

Testing and Tracing

No Libraries required however the log library and dgb! macro are very useful.

Make use of test annotations:

#[cfg(test)]
mod tests {
    #[test]
    fn exploration() {
        assert_eq!(2 + 2, 4);
    }

    #[test]
    fn another() {
        panic!("Make this test fail");
    }
}

We don't need a library for parallel tests just configure with:

cargo test -- --test-threads=1
