The Borrow Checker in Action

Rust keeps track owners of memory (That is a variable owns a piece of memory and has a lifetime during the program). During stages of execution that memory may be freed after use or borrowed during modification.

fn main(){
    let s = String::from("hello");
    borrow_and_print(s);
    // uncomment below to see the borrow checker in action
    //println!("{}", s);
}

// s is string stored on the heap and borrowed by this function
fn borrow_and_print(s: String) {
    println!("{}", s);
}

Awesome pattern matching syntax with match

From the Rust Book:

fn main () {
    for i in 1..=100 {
        match (i % 3, i % 5) {
            (0,0) => println!("FizzBuzz"),
            (0,_) => println!("Fizz"),
            (_,0) => println!("Buzz"),
            (_,_) => println!("{}", i)
        }
    }
}

Its all about the memory

  • Some languages have automatic collection of unused memory Java, NodeJS they keep track of references using memory
  • Others require manual memory management like C, C++ allocations and removal must be done manually, fast but error prone, a double deallocate is a bug, a missed deallocate is a memory leak, sometimes this is also misused as a feature
  • Rust uses a slightly different concept of ownership. Where variables are kept track by an owner and the value dropped once this single owner is out of scope

#![allow(unused)]
fn main() {
{                      // s is not valid here, it’s not yet declared
    let s = "hello";   // s is valid from this point forward

    // do stuff with s
}     
                        // this scope is now over, and s is no longer valid
}

What Rust does? A move instead of a shallow copy


#![allow(unused)]
fn main() {
let s1 = String::from("hello");
let s2 = s1;

println!("{}, world!", s1);
}

s2 is not a shallow copy or reference but the actual pointer to the previous s1 with a new name. The above will throw a compile time error preventing mistakes being made.

Deep copies are never made explicitly


#![allow(unused)]
fn main() {
let s1 = String::from("hello");
let s1 = String::from("bye");

println!("{}, world!", s1);
}

This is a shadowed var, enforced by the compiler it can help cleaner variable naming when you want to process data.

Parallel Programming

Rust natively supports parallel programming but we also have some cool things:

rayon simplified data parallelism with iterators in Rust. This automatically spins up threads on separate CPUs.


#![allow(unused)]
fn main() {
use rayon::prelude::*;

fn sum_of_squares(input: &[i32]) -> i32 {
    input.par_iter() // -> Parallel iterator
         .map(|&i| i * i)
         .sum()
}
}
