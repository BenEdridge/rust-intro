fizz buzzing 1-100 with Rust matchers

fn main () {
    for i in 1..=100 {
        match (i % 3, i % 5) {
            (0,0) => println!("FizzBuzz"),
            (0,_) => println!("Fizz"),
            (_,0) => println!("Buzz"),
            (_,_) => println!("{}", i)
        }
    }
}

Its all about managing memory

  • Some languages have automatic collection of unused memory Java, NodeJS they keep track of references using memory
  • Others require manual memory management like C, C++ allocations and removal must be done manually, fast but error prone, a double deallocate is a bug, a missed deallocate is a memory leak, sometimes this is also misused as a feature
  • Rust uses a slightly different concept of ownership. Where variables are kept track by an owner and the value dropped once this single owner is out of scope
    {                      // s is not valid here, it’s not yet declared
        let s = "hello";   // s is valid from this point forward

        // do stuff with s
    }     
                           // this scope is now over, and s is no longer valid

Note: In C++, this pattern of deallocating resources at the end of an item’s lifetime is sometimes called Resource Acquisition Is Initialization (RAII). The drop function in Rust will be familiar to you if you’ve used RAII patterns.

What Rust does? A move instead of a shallow copy

    let s1 = String::from("hello");
    let s2 = s1;

    println!("{}, world!", s1);

s2 is not a shallow copy but the actual pointer to the previous s1 with a new name.

This is very similar to the shadowing support in Rust. The above will throw a great compile time error preventing mistakes being made.

Deep copies are never made explicitly

    let s1 = String::from("hello");
    let s1 = String::from("bye");

    println!("{}, world!", s1);

This is a shadowed var, enforced by the compiler it can help cleaner code when you want to process data.

    let s1 = String::from("hello");
    let s2 = s1.clone();

    println!("{}, world!", s1);

The package manager cargo and package registry crates.io

cargo is the package manager for Rust that pulls in packages from crates.io and provides many similar functions to npm.

A simple registry can be implemented in Git.

cargo provides a number of commands for building, cleaning, installing and publishing packages.

Some of the commands we will be using are:

#Create a new cargo package
cargo init

#Run a binary or example of the local package
cargo run

#Build this package's and its dependencies' documentation
cargo doc

#Run the tests
cargo test
